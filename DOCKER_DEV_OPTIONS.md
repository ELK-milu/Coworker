# Docker 开发环境配置方案

## 方案对比

### 方案A：官方推荐方式（简单）

**架构：**
```
浏览器访问: http://localhost:5173 (前端开发服务器)
                ↓
        Vite Dev Server (5173)
                ↓
        API 代理 → Go 后端 (3000)
```

**优点：**
- ✅ 遵循官方推荐
- ✅ 配置简单，无需 Nginx
- ✅ Vite HMR 热重载更快
- ✅ 前后端完全独立开发

**缺点：**
- ❌ 开发环境访问端口（5173）与生产环境（3000）不同
- ❌ 需要记住两个不同的端口

---

### 方案B：统一端口方式（复杂）

**架构：**
```
浏览器访问: http://localhost:3000 (与生产环境一致)
                ↓
        Nginx 反向代理 (3000)
                ↓
        ┌───────┴───────┐
        ↓               ↓
    Vite (5173)    Go (3001)
```

**优点：**
- ✅ 开发和生产环境访问方式一致
- ✅ 更接近生产环境的架构

**缺点：**
- ❌ 配置复杂，多一层 Nginx
- ❌ 调试时需要查看多个容器日志
- ❌ 不是官方推荐的方式

---

## 推荐选择

**建议使用方案A（官方推荐）**，原因：
1. 官方文档明确说明这是推荐的开发方式
2. 配置更简单，出问题更容易排查
3. 前后端完全独立，开发体验更好
4. 生产环境和开发环境本来就应该有所区别

**如果必须使用方案B**，可以保留当前的 Nginx 配置。
